# 技术方案与开发步骤

## 技术栈选择

第一阶段采用 **纯 HTML/CSS/JavaScript PWA + Supabase**。

原因：
- 小白友好：不用先学习复杂构建工具。
- 手机电脑都能用：部署 HTTPS 后，iPhone Safari 可添加到主屏幕。
- 本地优先：数据先保存在浏览器，没网也能学习。
- 云同步成本低：Supabase 免费额度足够个人学习数据使用。
- 可逐步升级：后续需要 AI 批改、账号系统、真题库时，再引入后端服务。

## 项目模块

| 模块 | 当前阶段 | 说明 |
|------|----------|------|
| 总览 Dashboard | MVP | 展示考试倒计时、今日任务、学习统计 |
| 备考计划 Plan | MVP | 根据考试日期和每日时长生成阶段计划 |
| 词汇 Vocab | MVP | 卡片记忆、SM-2 间隔复习、进度统计 |
| 四科练习 Practice | 阶段 2 | 手动记录听说读写训练 |
| 错题本 Notebook | 阶段 2 | 记录错因、解决办法、复盘状态 |
| 模考 Mock | 阶段 2 | 记录 L/R/W/S 和 Overall |
| 刷题 Quiz | 阶段 3 | 真题题库、答题、判分、解析 |
| 阅读/听力专项 | 阶段 4-5 | Passage、音频、题目、错题入库 |
| 写作/口语 | 阶段 6 | 编辑器、录音、复盘，后续接 AI |

## 稳定推进步骤

1. 项目基座：目录、文档、PWA、本地运行、开发日志。
2. MVP：备考计划 + 今日任务 + 词汇记忆。
3. 数据闭环：练习记录、错题本、模考记录。
4. 同步闭环：Supabase 加密同步、iPhone PWA 验证。
5. 题库闭环：阅读/听力题目数据结构、答题页、判分。
6. 复盘闭环：错题自动入库、周报、计划自动调整。
7. 上线与备份：GitHub、免费托管、数据导出、版本检查。

## 开发纪律

- 每次只改当前模块相关文件。
- 修改 JS 后运行 `npm run check`。
- 开发结束更新 `CLAUDE.md` 和当天 `dev-logs/YYYY-MM-DD.md`。
- 新功能先本地验证，再考虑部署。
- 学习资料来源必须可追踪；涉及版权的真题资料不直接内置盗版内容。
